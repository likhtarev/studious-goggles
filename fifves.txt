В Wireshark можно наблюдать, как отличаются пакеты TCP и UDP. Вот основные различия между ними на основе анализа заголовков и их полей.

---
1. TCP (Transmission Control Protocol)

TCP — протокол с установкой соединения, ориентированный на надежную передачу данных.

Основные характеристики TCP:

1. Управление соединением:
   - TCP устанавливает соединение с использованием трехстороннего рукопожатия (3-way handshake). 
   - Для завершения соединения используется процесс с четырехэтапным завершением (4-way teardown).

2. Порядок доставки:
   - TCP обеспечивает последовательность доставки данных. Если пакеты приходят в неправильном порядке, TCP собирает их в правильной последовательности.

3. Поля заголовка TCP (видимые через Wireshark):
   - Checksum: Проверяет целостность данных. Это обязательное поле для TCP.
   - Sequence Number (SEQ): Указывает порядок данных для восстановления правильной последовательности.
   - Acknowledgment Number (ACK): Подтверждает получение данных от отправителя.
   - Flags (Control Bits): Включает такие флаги, как:
     - SYN: Начало соединения.
     - ACK: Подтверждение.
     - FIN: Завершение соединения.
     - RST: Сброс соединения.
     - PSH: Принудительная отправка данных.
     - URG: Указание приоритетных данных.
   - Window Size: Указывает размер окна для управления потоком.

4. Ретрансляция:
   - Если данные теряются, TCP отправляет их заново.

5. Скорость передачи:
   - TCP поддерживает управление потоком (Flow Control) и контроль перегрузки.

---

Пример анализа TCP-пакета через Wireshark:

- Протокол: TCP
- Source Port: 12345
- Destination Port: 80 (HTTP)
- Flags: SYN, ACK
- Sequence Number: Указывает начальный номер последовательности.
- Acknowledgment Number: Подтверждает получение пакета.
- Checksum: Проверка целостности.
- Window Size: Размер буфера данных.

---

2. UDP (User Datagram Protocol)

UDP — протокол без установки соединения, ориентированный на высокую скорость.

Основные характеристики UDP:

1. Управление соединением:
   - UDP не устанавливает соединение перед передачей данных. Пакеты передаются "как есть".

2. Порядок доставки:
   - UDP не гарантирует порядок доставки пакетов. Если пакеты теряются или приходят в неправильной последовательности, их обработка остается за приложением.

3. Поля заголовка UDP (видимые через Wireshark):
   - Checksum: Проверяет целостность данных. Это поле необязательное и может быть отключено.
   - Length: Указывает длину заголовка и данных.
   - Source Port: Порт отправителя.
   - Destination Port: Порт получателя.

4. Ретрансляция:
   - UDP не поддерживает автоматическую ретрансляцию потерянных данных.

5. Скорость передачи:
   - UDP быстрее TCP, так как не требует установки соединения и подтверждения доставки.
